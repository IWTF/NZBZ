### 1.1 开发背景

随着碎片化阅读的侵蚀，大家越来越缺乏静心读一本纸质书的动力和机会。之前报道德国、法国进行了电话亭改造项目，将其改造为小型图书馆，不仅利用了闲置资源，也充分利用了大家闲置的旧书。又比如之前流行一时的地铁传书，也受到大家的广泛好评。受这些活动的启发，我们小组决定开发一个图书漂流和分享小程序，让大家闲置的图书“游”起来，并依此建立书友圈，以书会友，将线下阅读体验与线上交流分享联系起来，激发全民阅读兴趣，更大地发挥纸质图书的价值。

除此之外，纸质书的多次传阅在某种程度上体现了环保理念，减少纸张消耗，减少原材料的砍伐。

### 2.1 需求分析

#### 2.1.1 功能需求

线下生活圈子中缺少交流分享的书友，电子产品带来的碎片化阅读挤掉人们阅读纸质书籍的时间，图书纸张消耗加重生态威胁……书鱼提供了现实的解决方案和平台。每个用户在平台上既是知识提供方，也是知识获取方，书圈内互相交流，互相督促，书圈外开阔眼界，更快地设计自己所不熟知的知识领域。更以积分、打卡的方式调动用户的积极性。

#### 2.1.2 非功能需求

* 系统后台保存用户部分个人信息，虽然由用户个人选择性填写，但需要确保信息数据不泄露，避免不法分子获取牟利。
* 系统在处理预约图书时，采用了小程序通知API，将预约者的地理位置信息发送至图书拥有者，除此之外不透露更多信息，保障预约者的人身安全。
* 系统实时监测每个用户的信誉情况，因为涉及到图书传送，如出现恶意传送其他物品，立即对账号处理，严重时与公安机关联系。

### 5.1 主要功能介绍&展示

#### 5.1.1首页

* 打开小程序首先进入首页。首页展示“我的标签”及实时更新的平台所有用户发表内容，畅游书海~

* 无论看中了哪本书，都可以点击该动态，跳转至详情页，显示这本书更加详细的内容信息，并进行收藏、分享等一系列操作。最重要的是，联系TA以成为这本书的下一个阅读者！通知TA时，我们通过小程序的通知API，直接发微信通知，并不泄露个人信息，保护用户隐私安全。最终书籍通过邮寄等方式到达您的手中。

  下图是“首页”的使用流程：

  ![流程图-首页.jpg](https://pic4.58cdn.com.cn/nowater/webim/big/n_v264ac5cd7a7d449e3803826afdb4fb868.jpg)

#### 5.1.2 书圈

* 书圈是书友及个性化推荐后显示的发表内容。这里是更加个性化的阅读交友圈，可以查看书籍热度排行、自己的书签、以及搜索好书，寻找志同道合的书友，发现更深更广阔的天地。进入自己的书签更有更换模板及生成海报功能，获取权限后图文海报保存进手机相册，成为更长久的纪念。

* 悬浮发表按钮，点击后进入发布笔记界面。发布内容包括图片、标题、新的笔记，并可设置可见范围，图书照片更可上传动图模式。在这里分享读书心得和笔记，畅所欲言~

* 与朋友圈类似，浏览者可点赞、评论、转发其他用户的笔记动态，使线下阅读与线上分享体验融为一体。

#### 5.1.3 我的

* 我的个人信息，包括我的分享、笔记、书友数量、积分等，其他信息可自行编辑修改。通过平台获得图书越多，积分越多，并设置每日阅读打卡，督促书友不断学习。

* 我的书架，方便用户查看个人收藏历史、书单、书友和笔记，对于用户读过的每一本书，这里都会生成它的漂流地图，记录它的来历、去向，之中蕴含的是每个读者的阅读故事。

* 更多有趣的快捷功能，如快递查询，切换主题，打开夜间模式等。
